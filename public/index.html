Available Routes:

api routes are accessed via '/api/v1', the 'v1'is to preserve the functionality of this current api while allowing for the

possiblity of creating a 'v2' of the api with different routes.


  • "/" : loads this static html file in place of the a frontend app
  • "/api/v1/physicians" : calls controller physicians/list.js to list all physicians
  • "/api/v1/physicians/get" : calls controller physicians/get.js to find specific physician by id/name
  • "/api/v1/physicians/create" : calls controller physicians/create.js to add a new physician to the db
  • "/api/v1/physicians/update" : calls controller physicians/update.js to update the info of selected physician(s)
  • "/api/v1/physicians/remove" : calls controller physicians/remove.js to delete a specific physician entry
  • "/api/v1/physicians/" : calls controller physicians/
  • "/api/v1/physicians/" : calls controller physicians/
